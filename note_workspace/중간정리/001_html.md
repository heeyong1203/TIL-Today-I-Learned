### 프론트엔드 개발자(Front-end Developer)
- 역할 : 사용자(클라이언트)가 직접 보는 화면을 구현하고, 서버와 통신하며 데이터 처리
- 주요 기술
  1) 마크업 언어: `HTML`
  2) 스타일: `CSS`
  3) 동작/로직: `JavaScript`
  4) 프레임워크: `React`, `Vue`, `Angular` 등
- 포커스: 사용자 인터페이스(UI), 사용자 경험(UX)

### 백엔드 개발자(Back-end Developer)
- 역할 : 사용자 눈에 보이지 않지만 서버에서 작동하는 기능 구현 (로그인, 데이터 저장 등)
- 주요 기술
  1) 언어: `Java`, `Pyton`, `Node.js` 등
  2) 데이터베이스: `MySQL`, `Oracle`, `MongoDB` 등
  3) 서버 프레임워크: `Spring`, `Express` 등
  4) API 설계 및 보안, 성능 최적화

### DevOps 엔지니어
- 역할 : 개발(Development)과 운영(Operation)의 중간에서 서버와 클라이언트를 효율적으로 연결
- 주요 기술
  * Linux, Docker, CI/CD, AWS, Nginx, Git 등
- 목표: 자동화, 배포 안정성, 서버 상태 모니터링

### 풀스택 개발자(Fullstack Developer)
- 역할 : 프론트엔드 + 백엔드 + 배포 전반을 아우르는 전천후 개발자
- 요구 기술: 위 기술들 대부분의 기본 이해, 팀워크, 실전 경험

  3. **JavaScript**: 화면 상에서 동작/이벤트 처리
  4. **Java**: 본격적인 백엔드 언어
  5. **SQL**: 데이터 저장과 조작 (MySQL, Oracle)
  6. **Spring Framework 등**: 백엔드 구조화와 실무 연습

# 🌐 HTML 개요 및 역사

### 1. HTML의 탄생 배경

HTML(HyperText Markup Language)은 **원래 웹페이지 제작용**으로 태어난 것이 아니라,  
초기에는 **과학자들끼리 문서를 컴퓨터 상에서 효율적으로 공유**하기 위한 목적으로 시작되었다.

- 1989년, **CERN(유럽 입자 물리 연구소)**에서 연구하던 컴퓨터 과학자 **팀 버너스리(Tim Berners-Lee)**가 HTML을 고안함
- 목표: 전 세계 과학자들이 **문서를 서로 연결(Link)**하고, **이메일처럼 공유**할 수 있는 체계 마련
- 당시 인터넷 환경은 있었으나, 웹은 없었고 "하이퍼텍스트" 개념만 존재했음


### 2. HTML의 개념

- **HTML은 프로그래밍 언어가 아니다**
  - 동작/논리를 처리하는 것이 아니라, **문서의 구조를 설명(마크업)** 하는 언어
- **HyperText Markup Language**의 약자
  - **HyperText**: 문서 간 연결(링크)
  - **Markup**: 태그를 이용해 문서의 구조를 표시
- 브라우저가 HTML을 해석하여 사용자에게 보기 쉬운 웹 페이지로 표시


### 3. HTML의 주요 특성

| 특성         | 설명 |
|--------------|------|
| 구조 설명 언어 | `<h1>`, `<p>`, `<img>` 등으로 문서의 구조를 정의함 |
| 정적 언어     | 단독으로는 동적인 동작이 없음 (JS와 연동해야 동적 페이지 가능) |
| 태그 기반     | `<tagname>내용</tagname>` 형태로 문서 구조를 표현 |
| 연결 가능     | `<a href="">` 태그로 다른 문서나 사이트로 연결 (HyperText의 핵심) |
| 브라우저 중심 | HTML 파일은 브라우저가 해석하고 출력하는 것이 일반적 |


### 4. HTML은 어떤 언어로 이루어졌는가?

- HTML 자체는 **"마크업 언어"**로, `<태그>`와 `속성`으로 구성됨
- 예시:

```html
<a href="https://www.example.com">이곳을 클릭하세요</a>
```

# 🧱 HTML 기본 구조와 주요 태그

---

### 1. 기본 태그
#### 태그(Tag)란?
- 문서를 원하는 서식대로 작성하기 위해서는 원하는 컨텐츠를 지정한 수단이 필요
- 따라서 HTML은 태그라 불리는 표기법으로 원하는 컨텐츠를 감싸서 영역을 지정
- 컨텐츠를 감싸서 영역을 지정하는 행위를 마크업(Mark up)이라 함 \
  `→ HTML은 마크업 기반의 언어`

#### 태그의 특징
1) 쌍으로 존재
    - `<p></p>`와 같이 여는 괄호와 닫는 괄호로 감싸야 함
    - 단 예외적인 태그들이 있음 (대체 태그, 빈 태그)
2) 중첩 가능
    - 태그 안에 태그를 포함시킬 수 있음
    - 바깥 쪽 태그를 부모, 내부 태그를 자식이라 함
3) 대소문자 구분 X
    - 프로그래밍 언어가 아니기 때문에, 대소문자를 구분하지 않음
    - 실제 업무에서는 대소문자를 섞어 사용하지 않고 오직 소문자로 작성
4) 태그 속성(Attribute)
    - 속성을 통해 추가적인 정보를 보유하고 능력을 확장할 수 있음
    - 속성은 시작 태그에만 작성할 수 있으며 속성="값" 형태로 작성해야 함


#### 대체 요소(Replacement Element)
- 태그 내부에 자체 컨텐츠를 보유할 수 없고, 외부 컨텐츠로 대체되는 태그
- 즉, 외부 매체를 불러오기 위한 용도의 태그

|요소|내용|
|-|-|
|`<img>`|이미지를 불러옴|
|`<video>`|동영상을 불러옴|
|`<audio>`|음성을 불러옴|
|`<iframe>`|웹페이지를 불러옴|


#### 빈 요소(Void Element)
- 닫는 태그 없이 스스로 종료되는 태그
- 내용을 포함할 수 없음
- html5 에서는 `</br>` 대신 `<br>`을 사용하는 것이 일반적 

|요소|내용|
|-|-|
|`<br>`|줄 바꿈 태그|
|`<hr>`|수평선 태그|
|`<link>`|외부 소스 링크 태그|

### 2. HTML 기본 문서 구조

HTML 문서는 기본적으로 다음과 같은 구조를 갖는다:

```html
<!DOCTYPE html>  <!-- HTML5 문서임을 선언 -->
<html lang="en"> <!-- lang 속성은 언어 설정 -->
<head>           <!-- head 속성은 문서의 정보(제목, 문자, 인코딩, 외부 파일 링크 등)-->  
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>           <!-- 사용자에게 실제 보여지는 웹 페이지 내용 -->
    <h1>여기가 본문</h1>
    <p>여기에 내용 작성</p>
</body>
</html>
```
